---
title: lpm ports
description: "List, kill, or reset dev-service port assignments."
---

```bash
lpm ports <action> [port]
```

Operates on the ports declared in `lpm.json > services`. Useful when a previous `lpm dev` run leaked a process holding a port, or when you want a quick view of which service maps to which number.

## Examples

```bash
lpm ports                # default — list ports declared in lpm.json
lpm ports list           # same
lpm ports kill 3000      # kill whatever's listening on :3000
lpm ports reset          # release LPM-managed reservations
```

## Subcommands

| Subcommand | Effect |
| --- | --- |
| (default) / `list` | List `services.<name>.port` entries from `lpm.json` |
| `kill <port>` | Kill the process listening on the given port (any process, not just LPM's) |
| `reset` | Release LPM-managed port reservations and clear stale state |

`list` is read-only. `kill` requires a port number — it errors out if you forget. `reset` is the "I have no idea why my dev server won't start" escape hatch.

## When to use each

- **`list`** — Quick check: "which service is on which port in this project?"
- **`kill <port>`** — Stale process from a crashed `lpm dev` is still bound. Faster than `lsof -i :3000` + `kill`.
- **`reset`** — You suspect LPM's own port-reservation state is stale. Clears it without touching the processes themselves.

## Flags

`lpm ports` takes no flags besides the [global flags](/docs/commands#global-flags).
