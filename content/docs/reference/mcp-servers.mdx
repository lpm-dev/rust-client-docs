---
title: MCP servers
description: "Wire Claude Code, Cursor, and Windsurf to LPM's MCP server."
---

```bash
lpm mcp <action> [name]
```

Manages [MCP](https://modelcontextprotocol.io) (Model Context Protocol) server configuration for AI editors. `lpm mcp setup` writes the LPM MCP server entry into your editor's MCP config so the editor's AI agent can call LPM tools (search, install hint, audit, etc.) on your behalf.

## Examples

```bash
lpm mcp setup                      # detect installed editors and configure them all
lpm mcp setup claude-code          # configure a specific editor
lpm mcp setup cursor
lpm mcp setup windsurf
lpm mcp status                     # which editors have LPM's MCP server wired up
lpm mcp remove cursor              # remove from one editor
```

## Subcommands

| Subcommand | Effect |
| --- | --- |
| `setup [name]` | Add LPM's MCP server entry to one or every detected editor's MCP config. Idempotent — safe to re-run. |
| `remove <name>` | Remove the LPM entry from a specific editor's config |
| `status` | List which editors have LPM's MCP server configured |

## Supported editors

| Editor | Config file |
| --- | --- |
| Claude Code | `~/.claude.json` (key: `mcpServers`) |
| Cursor | `~/.cursor/mcp.json` (key: `mcpServers`) |
| Windsurf | `~/.codeium/windsurf/mcp_config.json` (key: `mcpServers`) |

`lpm mcp setup` (without a name) walks every editor whose config file is detected and adds the entry. Editors that aren't installed are skipped silently.

## What gets written

The entry registers an MCP server pointing at LPM's bundled server binary. No additional installation required — the binary ships inside the LPM CLI.

After `setup`, restart the editor and ask the agent to e.g. "find a JSON-schema validator" — it'll route through the MCP tools to LPM's search and quality endpoints rather than guessing from training data.

## Flags

`lpm mcp` takes no specific flags besides the [global flags](/docs/commands#global-flags).
