---
title: lpm exec
description: "Run a JS or TS file directly — no scripts entry, no transpile step."
---

```bash
lpm exec <file> [-- args...]
```

Runs a single JavaScript or TypeScript file. The runtime is auto-detected from the extension: `.js` / `.mjs` / `.cjs` go through `node`; `.ts` / `.tsx` go through `tsx` (zero-config TypeScript loader).

Useful for one-off scripts (`scripts/seed.ts`, `tools/migrate.js`) without adding a `scripts` entry to `package.json` and without spinning up `ts-node` / `tsx` yourself.

## Examples

```bash
lpm exec scripts/seed.ts
lpm exec scripts/migrate.js
lpm exec scripts/seed.ts -- --env=staging      # forward args to the script
lpm exec src/cli.ts arg1 arg2                  # bare trailing args also work
```

## How runtime is picked

| Extension | Runtime |
| --- | --- |
| `.js`, `.mjs`, `.cjs` | `node` |
| `.ts`, `.tsx` | `tsx` (resolved from `node_modules/.bin` or shipped with LPM) |

## Argument forwarding

Anything after the file is forwarded to it. `--` is optional but useful when you want to pass args that look like LPM flags:

```bash
lpm exec scripts/seed.ts -- --verbose --env=prod
```

## Flags

| Flag | Effect |
| --- | --- |
| `--no-env-check` | Skip env-var schema validation (matches `lpm run --no-env-check`) |

Plus the [global flags](/docs/commands#global-flags).
