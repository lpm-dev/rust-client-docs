---
title: lpm use
description: "Install and pin a Node.js version for the project. Also: manage env vars."
---

```bash
lpm use <runtime>@<version>      # install + pin
lpm use --list                   # list installed versions
lpm use vars <subcommand>        # manage encrypted env vars
```

Installs and pins a runtime version. Today, only `node` is supported — additional runtimes (`deno`, `bun`) are reserved for future versions.

After `lpm use node@22`, scripts run with the pinned Node version via PATH injection. The pin is recorded in `lpm.json` so teammates and CI pick up the same version.

`lpm use vars` is a separate subcommand for managing encrypted env vars — see below.

## Installing a Node version

```bash
lpm use node@22                # install + pin Node 22 (resolves to latest 22.x)
lpm use node@lts               # install + pin the latest LTS
lpm use node@22.12.0           # install + pin an exact version
lpm use node@22 --pin          # already installed — just pin, no install
```

Versions land under `~/.lpm/runtimes/node/`. `lpm use node@22` resolves the spec, downloads the build for your platform if missing, then writes the pin to `lpm.json`:

```json title="lpm.json"
{ "runtime": { "node": "22.12.0" } }
```

`.nvmrc` is honored as a fallback when `lpm.json > runtime.node` is absent.

## Listing installed versions

```bash
lpm use --list                 # list installed Node versions
lpm use node --list            # same — runtime arg is optional
```

## Env vars (`lpm use vars`)

The `vars` sub-action manages project-level environment variables, optionally per-environment (`--env=staging`).

```bash
lpm use vars list                              # list all vars (values masked)
lpm use vars list --reveal                     # list with values shown
lpm use vars get DATABASE_URL                  # get one var
lpm use vars get DATABASE_URL --reveal         # get with value shown
lpm use vars set DATABASE_URL=postgres://...   # set one
lpm use vars set FOO=bar BAZ=qux               # set many at once
lpm use vars delete FOO BAZ                    # delete one or more
lpm use vars import .env.production            # import from a file
lpm use vars import .env --overwrite           # overwrite existing values
lpm use vars export .env.backup                # export to a file

lpm use vars set --env=staging API_URL=https://staging.example.com
lpm use vars list --env=staging
```

`vars` works on top of LPM's secrets infrastructure — see [Secrets vault](/docs/infra/secrets-vault) for the storage and encryption model.

## Flags

| Flag | Effect |
| --- | --- |
| `--list` | List installed versions for the runtime |
| `--pin` | Pin only (skip install if the version is already present) |

Plus the [global flags](/docs/commands#global-flags).
