---
title: lpm use
description: "Install and pin a Node.js version for the project."
---

```bash
lpm use <runtime>@<version>      # install + pin
lpm use --list                   # list installed versions
```

Installs and pins a runtime version. Today, only `node` is supported — additional runtimes (`deno`, `bun`) are reserved for future versions.

After `lpm use node@22`, scripts run with the pinned Node version via PATH injection. The pin is recorded in `lpm.json` so teammates and CI pick up the same version.

For project environment variables and secrets management, see [`lpm env`](/docs/dev/env).

## Installing a Node version

```bash
lpm use node@22                # install + pin Node 22 (resolves to latest 22.x)
lpm use node@lts               # install + pin the latest LTS
lpm use node@22.12.0           # install + pin an exact version
lpm use node@22 --pin          # already installed — just pin, no install
```

Versions land under `~/.lpm/runtimes/node/`. `lpm use node@22` resolves the spec, downloads the build for your platform if missing, then writes the pin to `lpm.json`:

```json title="lpm.json"
{ "runtime": { "node": "22.12.0" } }
```

`.nvmrc` is honored as a fallback when `lpm.json > runtime.node` is absent.

## Listing installed versions

```bash
lpm use --list                 # list installed Node versions
lpm use node --list            # same — runtime arg is optional
```

## Flags

| Flag | Effect |
| --- | --- |
| `--list` | List installed versions for the runtime |
| `--pin` | Pin only (skip install if the version is already present) |

Plus the [global flags](/docs/commands#global-flags).
