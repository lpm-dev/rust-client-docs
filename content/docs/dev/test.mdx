---
title: lpm test
description: "Auto-detect and run the project's test runner — vitest, jest, or mocha."
---

```bash
lpm test [-- args...]
```

Detects which test runner is installed in the project (vitest → jest → mocha) and runs it, forwarding any trailing arguments verbatim. No config; no per-runner subcommand.

## Examples

```bash
lpm test                              # run the whole suite
lpm test src/utils.test.ts            # run one file
lpm test -- --run                     # vitest: single-pass mode
lpm test -- --watch                   # forward --watch
lpm test -- --reporter=verbose
```

## Detection order

1. `vitest` (in `dependencies` or `devDependencies`)
2. `jest`
3. `mocha`

Whichever is found first wins. If none is installed, `lpm test` reports it.

## Argument forwarding

Anything after the command (or after `--`) is forwarded to the runner:

```bash
lpm test -- --coverage --reporter=html
# vitest: vitest --coverage --reporter=html
# jest:   jest   --coverage --reporter=html
```

## Flags

`lpm test` has no LPM-specific flags — everything is forwarded to the underlying runner. Use the [global flags](/docs/commands#global-flags) for output formatting (`--json`, `--verbose`).
