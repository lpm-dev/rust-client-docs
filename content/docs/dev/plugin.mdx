---
title: lpm plugin
description: "Manage built-in tool plugins — list, update, and remove lazy-downloaded binaries."
---

```bash
lpm plugin <action> [name]
```

Manages the lazy-downloaded binaries that back [`lpm lint`](/docs/dev/lint) (Oxlint), [`lpm fmt`](/docs/dev/fmt) (Biome), and friends. Plugins are downloaded on first use, cached under `~/.lpm/cache/plugins/`, and verified against published SHA-256 checksums.

## Examples

```bash
lpm plugin list                # show installed plugins, current vs latest
lpm plugin update              # update every plugin to its latest version
lpm plugin update oxlint       # update one plugin
lpm plugin remove biome        # delete cached binaries for one plugin
```

## Available plugins

| Plugin | Backs | Notes |
| --- | --- | --- |
| `oxlint` | [`lpm lint`](/docs/dev/lint) | Distributed as `.tar.gz` archives by `oxc-project/oxc` |
| `biome` | [`lpm fmt`](/docs/dev/fmt) | Distributed as direct binaries by `biomejs/biome` |

`lpm check` is **not** a plugin — it shells out to the project-local `tsc`.

## Pinning per-project

```json title="lpm.json"
{
  "tools": {
    "oxlint": "1.57.0",
    "biome": "2.4.8"
  }
}
```

When a pinned version isn't in the cache, the next `lpm lint` / `lpm fmt` downloads and verifies it. Pinned versions don't have shipped checksums — verification is skipped with a warning.

## Subcommands

| Subcommand | Aliases | Effect |
| --- | --- | --- |
| `list` | `ls` | Print installed plugins, the current version, and the latest available |
| `update` | `upgrade` | Pull the latest release for one plugin (or all if no name given) |
| `remove` | `rm`, `uninstall` | Delete cached binaries for one plugin |

Plus the [global flags](/docs/commands#global-flags) — `--json` returns a structured listing.
