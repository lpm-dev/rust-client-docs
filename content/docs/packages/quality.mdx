title: lpm quality
description: "Show the quality report for a package — score, tier, and per-check breakdown."
---

```bash
lpm quality <package>
```

Fetches the registry's quality report for a package and prints it: a score out of 100, a tier label, and a per-check breakdown grouped by category (readme, license, types, tests, maintenance, etc.).

The same report runs locally during [`lpm publish`](/docs/packages/publish) — `lpm quality` lets you preview a package's grade without publishing or installing.

## Examples

```bash
lpm quality zod
lpm quality @lpm.dev/owner.pkg
lpm quality zod --json | jq '.checks[] | select(.passed == false)'
```

## What's shown

- **Score** — points out of max (color-coded)
- **Tier** — human label (e.g., `excellent`, `good`, `passing`, `needs-work`)
- **Ecosystem** — `js`, `swift`, etc.
- **Checks** — per-category list with pass/fail icons, label, and points awarded

## Flags

`lpm quality` takes no specific flags besides the [global flags](/docs/commands#global-flags). `--json` returns the full report including check IDs, descriptions, and remediation hints.
