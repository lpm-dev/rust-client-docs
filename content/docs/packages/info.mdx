---
title: lpm info
description: "Show metadata for a package — versions, dependencies, integrity, ecosystem."
---

```bash
lpm info <package> [--version <V>]
```

Fetches a package's metadata from the registry and prints it in a human-readable card: name, description, the chosen version's ecosystem, integrity hash, and full dependency list. With `--json`, returns the raw metadata for tooling.

By default, `lpm info` shows the latest version. Pass `--version <V>` to inspect a specific release.

## Examples

```bash
lpm info zod
lpm info zod --version 4.3.6
lpm info @lpm.dev/owner.pkg --version 2.1.0
lpm info react --json | jq '.versions["19.0.0"].dependencies'
```

## What's shown

For the chosen version:

- `version` — the actual semver
- `ecosystem` — `js`, `swift`, etc.
- `integrity` — SRI hash (truncated for display, full in `--json`)
- `dependencies` — declared runtime deps with their ranges

The full registry payload (every version, dist-tags, maintainers, README, etc.) is available via `--json`.

## Flags

| Flag | Effect |
| --- | --- |
| `--version <V>` | Show a specific version instead of latest |

Plus the [global flags](/docs/commands#global-flags) — `--json` returns the full registry record.
