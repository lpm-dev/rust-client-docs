---
title: lpm outdated
description: "List dependencies with newer versions available."
---

```bash
lpm outdated
```

Compares every entry in `package.json` against the registry and prints the ones with newer versions available. Read-only — never modifies anything.

## Example output

```text
Package           Current   Wanted    Latest
react             18.2.0    18.3.1    19.0.0
zod               4.3.5     4.3.6     4.3.6
typescript        5.4.0     5.4.5     5.6.2
```

| Column | Meaning |
| --- | --- |
| **Current** | What's resolved in your lockfile right now |
| **Wanted** | The newest version that satisfies your declared range |
| **Latest** | The newest version on the registry, regardless of range |

A row appears only when at least one of `Current` < `Wanted` or `Current` < `Latest` is true.

## Acting on the report

```bash
lpm upgrade           # bump to "Wanted" (stays within ranges)
lpm upgrade --major   # also offer "Latest" (cross-major bumps)
```

For a single package:

```bash
lpm install zod@latest
```

## Flags

`lpm outdated` takes no flags besides the [global flags](/docs/commands#global-flags) (`--json` is especially useful for scripting).
